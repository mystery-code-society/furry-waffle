HELLO, CODERS!

Let's get set up for our projects.

This is a multistep process, as follows:

  1. Sign Up For GitHub and Cloud9 Accounts
  2. Add Your SSH key to GitHub
  3. Create a Cloud9 Workspace
  4. Serve a Web Page
  5. git - Pushing Your First Commit
  6. Set Your GitHub Pages Branch
  7. Host an Image
  8. Add a Stylesheet
  9. Commit and Push Your Latest Changes

Step 1: Sign Up For GitHub and Cloud9 Accounts

Go to github.com and sign up for an account

  • Your username will be in your project URL, so pick a good handle for yourself
  • Use an email address that you have access to during class such as gmail
  • Create a secure password following GitHub's guidelines

Create a New Repository

  • Click the + in the top menu and select "New repository"
  • Your repository name is also in the project URL, so choose carefully
  • Description is optional, you can leave it blank
  • Leave the default setting as a Public project (free GitHub accounts don't have Private projects)
  • Leave the default settings with no README, .gitignore: None, license: None

Accept your invitation to Cloud9

(You should have received an email)

  • Follow the link in the email to sign up for our team on Cloud9
  • To sign into your account next time, you'll go to https://c9.io/login

Step 2: Add Your SSH Key to GitHub

Get your Cloud9 SSH Key

  • Click the gear icon to go to Settings
  • Select SSH Keys on the side menu
  • You want the SSH key for your private git repository on the bottom
  • Select the really long string of characters and copy it to your clipboard

Enter your SSH key into GitHub

  • Click your profile to go to your GitHub account settings
  • Select SSH and GPG keys from the side menu
  • Click the green New SSH key button
  • Give your SSH key a memorable title like Cloud9
  • Paste the really long string of characters from Cloud9 into the text area and click Add SSH key
  • Your new SSH key should now be listed

Step 3: Create a Cloud9 Workspace

Create a new workspace

  • Click the + icon to create a new workspace
  • Enter a memorable workspace name with no spaces that is relevant to your project
  • Select the HTML5 template

Link your Git repository

  • Wait! Before clicking that exciting, green Create workspace button in Cloud9
  • Paste your GitHub repository URL into the Clone from Git URL text box
  • Now that you've entered that GitHub url, click Create workspace in Cloud9
  • Wait just a bit while your workspace is spun up
  • Maybe grab a snack or take a selfie cuz it's about to get real!

Step 4: Serve a Web Page

Create a New HTML File from Template

  • Select New From Template -> HTML File from the File menu
  • File -> Save As and give it the filename "index.html"

Run the Web Server

  • Open the index.html file
  • Type something in the body of the HTML document
  • Save the changes to the index.html file with File -> Save or Ctrl+S
  • Click the Run button to start the Web Server
  • Click the url in the server log output in the terminal
  • Select Open to show the website in a new tab
  • Dismiss the warning and click Open the App
  • View your website in all its glory

Step 5: git - Pushing Your First Commit

Check the status of changes

  • In the lower section of the Cloud9 IDE, click a tab that starts with "bash"
  • Or if you don't see a bash tab, click the + button then click New Terminal
  • On the command line, type:
git status

Add changed files to the commit

  • On the command line, type:
git add -A

Create a commit with the message "first commit"

Commits allow us mark where we introduced specific changes into our version history

  • On the command line, type:
git commit -m "first commit"

Push the commit up to the remote repository (GitHub)

  • On the command line, type:
git push origin master

Step 6: Set Your GitHub Pages Branch

Project Settings

  • Bring up your project on GitHub and navigate to the Settings tab

GitHub Pages Branch

  • In Settings, scroll down to the GitHub pages section
  • Select the master branch and click Save

GitHub Pages URL

  • Your GitHub Pages website url will be shown in a blue banner
  • The URL format is https://username.github.io/project/

View your website on the live internet!

  • Click the GitHub Pages url link to open your site

Step 7: Host an Image

Create a folder for images

  • Right click your top-most project folder in the left section listing files and folders
  • Select New Folder from the context menu
  • Name the folder "images"

Add an image file to the folder

  • Drag and drop an image file from your local computer onto the images folder in Cloud9
  • Add an img tag to the body of your index.html document

View the image on your development site

  • Click the url in the server log output in the terminal
  • Select Open to show the website in a new tab
  • (or just refresh the development site tab if you left it open earlier)

Step 8: Add a Stylesheet

Add a CSS file

  • Add a New Folder to your project called "css"
  • Right click the new css folder and select New File
  • Name the file "style.css"
  • Add some css to the style.css file, such as setting the background-color on the body element to pink
  • Save the changes to the style.css file with File -> Save or Ctrl+S

Add a link to the stylesheet

  • In the head of your index.html page, add a link tag to include the stylesheet
  • Save the changes to the index.html file with File -> Save or Ctrl+S
<link rel="stylesheet" type="text/css" href="css/style.css">

Check out your site!

  • Click the url in the server log output in the terminal
  • Select Open to show the website in a new tab
  • (or just refresh the development site tab if you left it open earlier)

Step 9: Commit and Push Your Latest Changes

  • Repeat Step 5, to check status, add files, commit, and push your changes
  • Use "add image and stylesheet" instead of "first commit" as the commit message
  • Changes may take a few minutes to sync, but they should be on your live GitHub Pages site shortly after you push
  • Visit your GitHub Pages site at https://username.github.io/project/
  • You may have to do a "hard refresh" in your browser with Ctrl+F5 to bypass cache
git status
git add -A
git commit -m "add image and stylesheet"
git push origin master
